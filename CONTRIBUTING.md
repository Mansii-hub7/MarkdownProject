# Contributing Guidelines

Thank you for your interest in contributing to the MarkdownProject!

## Code Style
* Please keep the Markdown clean and readable.
* Use headers (#) for structure.
* Check for spelling errors before submitting.

## Pull Requests
1. Fork the repository.
2. Create a new branch for your feature (`git checkout -b feature/AmazingFeature`).
3. Commit your changes.
4. Open a Pull Request.

## Reporting Issues
* Use the "Issues" tab on GitHub.
* Be descriptive and provide screenshots if possible.
